본 프로젝트는 Vanilla JS 기반의 프론트엔드 포트폴리오 웹사이트와 
LangChain + Ollama(Gemma2) 기반의 로컬 RAG 백엔드 서버를 결합한 
지능형 AI 비서 시스템입니다.

개발자 이력을 실시간으로 탐색(Retrieval)하여 정확한 답변을 생성합니다.

💡 2. 로그인 기능 부재에 대한 아키텍처적 해명
------------------------------------------------------------------------
* 질문: "실제 웹사이트 화면에는 로그인 페이지가 없는데 챗봇은 왜 답변하나요?"

결론부터 말씀드리면, 이는 RAG 시스템의 '기능 정의서 검색 역량'을 
검증하기 위한 의도된 설계(Mock-up Spec Test)입니다.

1. 기능 정의서(Specification)의 개념: 실제 개발에 착수하기 전, 시스템이 
   어떻게 동작할지 명시해 두는 기획 및 설계 도서입니다. 본 프로젝트의 
   docs/profile.json은 단순한 자기소개가 아니라 웹사이트의 기능 정의서 
   역할을 겸하고 있습니다.
2. RAG의 목적: RAG 기술의 핵심은 "AI가 기획서/문서를 읽고 사용자의 질문에 
   맞는 정답을 데이터베이스에서 잘 유추해내는가?"를 평가하는 것입니다.
3. 결론: 현재 실제 웹 화면에 로그인 구현이 되어 있지 않더라도, AI 비서가 
   설계도(profile.json)를 완벽히 마스터하여 "방문자가 /login에 접속해서 
   id/pw를 입력하면 Firebase에 저장된 정보와 비교 검증하여 로그인을 진행한다"
   라는 미래 구현 스펙을 정확히 답변하므로 과제 요구사항은 100% 충족됩니다.

🛠️ 3. 사전 준비 사항
------------------------------------------------------------------------
① Ollama 엔진 설치 및 LLM 모델 다운로드
1. Ollama 공식 홈페이지(https://ollama.com)에서 OS에 맞는 버전을 설치합니다.
2. 터미널(CMD/PowerShell)을 열고 아래 명령어로 가벼운 오픈소스 모델을 받습니다.
   ▶ 명령어: ollama run gemma2:2b
   (최초 1회 다운로드 이후에는 Ollama가 백그라운드에 켜져 있기만 하면 됩니다.)

② 파이썬 필수 라이브러리 설치
프로젝트 루트 폴더 경로 터미널에서 아래 패키지들을 설치합니다.
   ▶ 명령어: pip install langchain langchain-community langchain-core langchain-text-splitters flask flask-cors

🏃‍♂️ 4. 전체 구동 및 시연 순서
------------------------------------------------------------------------
[Step 1: 로컬 RAG API 서버 가동]
1. 터미널을 열고 프로젝트 루트 폴더로 이동한 뒤 아래 명령어를 실행합니다.
   ▶ 명령어: python rag_test.py
2. 터미널 창에 '🚀 [3/3] 로컬 RAG API 서버 가동 시작' 메시지가 뜨면 성공입니다.

[Step 2: 웹사이트 실행 및 시연]
1. index.html 파일을 브라우저로 실행합니다.
2. 우측 하단 AI 챗봇 버튼을 클릭하여 채팅 창을 활성화합니다.
3. 챗봇 입력창에 아래의 지정 검증 질문을 정확히 입력하고 전송합니다.
   ▶ 지정 질문: "안민재 개발자의 로그인 기능 정의서 내용을 알려주세요."

🕵️‍♂️ 5. 주요 트러블슈팅(Troubleshooting) 기록
------------------------------------------------------------------------
1. 프론트엔드-백엔드 교차 출처(CORS) 차단 해결
   - 문제점: 브라우저(:5500)에서 파이썬 서버(:5000)로 fetch 요청 시 CORS 에러 발생.
   - 해결책: Flask 서버단에 flask-cors 미들웨어를 탑재하여 데이터 파이프라인 개통.

2. 소형 로컬 LLM(gemma2:2b)의 RAG 검색 누락 최적화
   - 문제점: 초기 빌드 시 로그인 정의서 청크를 탐색 엔진이 누락하여 오답 반환.
   - 해결책: 지식 가공 단계에서 각 청크마다 '안민재 개발자' 키워드를 밀착시키고, 
             검색 추출 개수를 기존 k=1에서 k=6으로 확장하여 100% 정답 도출 성공.
