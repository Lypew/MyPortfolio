주요 프로젝트

타임캡슐 보이스

STT (Whisper-Turbo) LLM (DNA-2.0-14B) TTS (Qwen3-TTS) Python

📌 한 줄 소개: 사용자의 음성을 Whisper-Turbo 모델로 실시간 인식(STT)하고, 한국어 특화 LLM이 답변을 생성(DNA-2.0)한 뒤, 이를 학습된 고유의 목소리로 변환(Qwen3-TTS)하여 출력하는 엔드투엔드(End-to-End) 음성 기반 인터랙티브 AI 에이전트 프로젝트입니다.

🔍 발견한 문제점 (Problem)

1. 베이스 모델의 언어 편향 문제: 초기 아키텍처 설계 당시 글로벌 베이스 모델을 LLM으로 채택했으나, 한국어 대화 흐름 도중 문맥에 맞지 않는 중국어 표현이 빈번하게 출력되어 대화의 몰입도를 깨뜨리는 현상이 발생했습니다.

2. TTS 텍스트 정규화(G2P) 오류: 음성 변환 과정에서 숫자 부호("0", "1" 등)를 문맥에 맞는 자연스러운 한국어(공, 일, 하나 등)로 읽지 못하고 타 언어식 발음으로 엉뚱하게 낭독하여 음성 답변의 직관성이 크게 저하되는 문제를 포착했습니다.

⚙️ 원인 분석 및 해결 (Solution)

1. 한국어 특화 LLM 교체 및 파이프라인 최적화: 단순한 프롬프트 엔지니어링만으로는 모델 자체의 언어 편향을 해결하기 어렵다고 판단하여, 한국어 언어 이해도와 성능이 검증된 국산 오픈소스 LLM인 `dnotitia/DNA-2.0-14B`로 전격 교체했습니다. 더불어 STT 단계에서 `Whisper-Turbo` 모델을 연계하여 전체 음성 파이프라인의 지연 시간(Latency)을 최소화하고, 100% 자연스러운 한국어 실시간 대화 플로우를 확보했습니다.

2. 데이터 전처리 딕셔너리 구축을 통한 발음 디버깅: TTS 엔진이 원시 숫자 데이터를 그대로 받아들일 때 발생하는 정규화 오류임을 분석해 낸습니다. 이를 해결하기 위해 텍스트가 TTS 모듈로 진입하기 직전 단계에서 숫자 패턴을 감지하여 정형화된 한국어 텍스트(`"0" -> "공"`, `"1" -> "일"`)로 강제 치환해 주는 전처리 파이프라인(G2P 보완 로직)을 직접 구현하여 발음 오류를 완벽히 해결했습니다.
